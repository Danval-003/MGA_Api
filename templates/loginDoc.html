API Lotes - Ruta /login

Iniciar sesión

URL: /login

Método: POST

Parámetros:

  • user (opcional): Nombre de usuario.
  • password (requerido): Contraseña del usuario.

Respuesta: Devuelve el resultado de la autenticación en formato JSON.

Ejemplo de solicitud:

POST /login
{
  "user": "john_doe",
  "password": "password123"
}

Ejemplo de respuesta exitosa:

{
  "message": "Autenticación exitosa",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxMjM0NTY3ODkwLCJpYXQiOjE2MzEyMjM2OTEsImV4cCI6MTYzMzA0ODg5MX0.2g1aEgzmZVuH3TlQ7FsUKAY2rOo7lrlMfiwTkSpnH2I"
}

Ejemplo de respuesta de error:

{
  "message": "Error al colocar los datos. Debes mandar 'user' y 'password' como parámetros"
}

Autentificacion de usuario

URL: /getUser

Método: GET

Respuesta: Verifica existencia del usuario ingresado.

Ejemplo de respuesta exitosa:

{
  Muestra de datos importantes del usuario
}

Ejemplo de respuesta de error:

{
  "message": "Usuario no autenticado"
}

Autentificacion del tipo de trabajador

URL: /getWorker

Método: GET

Respuesta: Verifica existencia del tipo de trabajador ingresado.

Ejemplo de respuesta exitosa:

{
  "message": "exito"
}
